# Reference-file provenance

Two reference files are load-bearing in the pattern library today:

## `conv1x1_passing_reference.v`

- **Source**: this repo, generated by Foundry on an earlier run and
  promoted to a reference after it passed end-to-end in the historical flow.
  Same license as the rest of this repo.
- **Used by**: `02_conv1x1.md` tells Foundry to adapt this file for any
  pointwise (1×1) conv2d. Treat its control structure as a proven template;
  new Artix-7/Vivado variants should use synchronous ROM reads and BRAM
  attributes instead of copying legacy async memory reads blindly.

## (planned) split-module spatial conv references

Spatial conv (KH*KW > 1) is being migrated to a split-module
architecture — `coord_scheduler` + `line_buf_window` + `conv_datapath` —
under `rtl_library/`. There is no monolithic `conv3x3_passing_reference.v`
today. See `rtl_library/SPLIT_ARCHITECTURE.md` for the contract.

## Rule for adding future references

Every file in `knowledge/references/` must be actually read by
`foundry.md`'s catalog and actually referenced by a `knowledge/patterns/*.md`
file for adaptation. If neither is true, delete the file — provenance-only
references are noise that mislead both reviewers and the agent.

When we do add a new reference (e.g. a future `conv3x3_passing_reference.v`),
record here: source repo, license, which pattern file consumes it.
