# MobileNetV2 — Engine TOP P=1 per-dispatch correctness verdict

**Date:** 2026-06-02
**Scope:** All 34 engine-dispatched pointwise (1×1) convolutions of MobileNetV2
(`node_conv_814 … node_conv_912`), each run in isolation through the **REAL**
`shared_engine` + real weight/bias/scale maps and compared byte-for-byte against
the per-module golden output vector.

## VERDICT

**The MobileNetV2 ENGINE TOP is byte-exact at P=1: 34 / 34 PASS, 0 FAIL.**
Total mismatch = 0, max|err| = 0 across every dispatch. No failure, no
root-cause needed.

Because all 34 dispatches are byte-exact, **the engine-based <80% fit artifact
is CORRECTNESS-PROVEN at the dispatch level.** Caveat: the full engine-top e2e
*value* sim is still pending; however the per-dispatch proof plus the fact that
the inter-dispatch plumbing (bram_to_stream bridges, activation loaders, the
shared activation/weight memories) is purely structural makes this a strong
result. The all-spatial e2e baseline (`mbv2_e2e_baseline_fresh.log`) stubs the
engine out, so this harness is the first and only real-engine mbv2 validation.

## How it was verified (latency-faithful)

The engine has a hardwired **WEIGHT_RD_LATENCY = 2** (2-cycle URAM read);
`mac_valid_in` is gated by `ag_weight_rd_en_d2` and the requant drain uses
`ag_mac_done_d5`. Every dispatch below was built with **WLAT=2** (the
deployment 2-cycle URAM read; `iso_uram_bank` `rd_data = r2`), **not** the
false-confidence 1-cycle path. This matters: history proved a 1-cycle iso
harness gives FALSE byte-exact confidence (`project_e2e_value_verification` —
WLAT=1 on conv_816 = mismatch 1,152,229/1,204,224, max|err|=252; WLAT=2 =
mismatch 0). The engine is instantiated `WGT_W=8` / `URAM_DATA_W=2048` (mbv2
stores INT8 byte weights, 32 OC/bank; OC k → byte k of the 2048-bit bus =
bank(k//32) byte-slot(k%32); requant is per-OC from `scale.mem`
`[15:0]=mult [21:16]=shift`, unconditional +HALF rounding + INT8 saturate — the
AXI scalar `scale_mult`/`shift` are vestigial).

## Harness

- `tb/engine_iso_wrap_mbv2.v` — wrapper: REAL `shared_engine` + 8 `iso_uram_bank`
  (288-bit line, low 256 used, READ_LATENCY=2 mirror of `nn2rtl_top.v`) +
  `iso_bias_mem`(bias/scale) + `iso_act_mem`.
- `tb/engine_iso_wrap_mbv2_tb.cpp` — AXI config + act preload + start + compare.
- `scripts/gen_engine_iso_cfg.py` — per-dispatch cfg → `tb/engine_iso_cfg.h`.
- `scripts/build_engine_iso_mbv2.bat 2` — Verilator build (the `2` selects WLAT=2).
- Engine RTL: `output/rtl/shared_engine_skeleton.v` +
  `output/rtl/engine/{address_generator,config_register_block,mac_array,requant_pipeline,bram_to_stream_bridge}.v`.
- Maps: `output/mobilenet-v2/weights/{uram_weights_bank0..7.mem,bias.mem,scale.mem}`.
- Goldens: `output/mobilenet-v2/goldens/node_conv_*.{goldin,goldout}`.

**Race note (load-bearing for reproduction):** the shared `tb/engine_iso_cfg.h`
is concurrently regenerated by a parallel workflow, so a naive gen→build→run can
silently bake the WRONG dispatch. The C++ `#include "engine_iso_cfg.h"` resolves
to the .cpp's own directory first, so `-I` ordering alone does not win. The
race-immune recipe used for the last 11 dispatches here:
`scripts/gen_iso_p1_cfg.py <mod>` writes the header into a **private dir**
(`tb_iso_p1/`, which also holds a copy of the wrapper .v and the tb .cpp), and
`scripts/build_iso_p1.bat <tag>` builds serial (`-j 1`) into a private
`obj_dir_iso_p1_<tag>_w2` with `scripts/nopch_override.mk` to defeat the
w64devkit precompiled-header race. The built `.exe` is `strings`-verified to
contain the target module id before trusting the run.

### Coverage of the historically-suspect failure modes

All four common engine bug classes are RULED OUT for every dispatch:

- **Stale weight (2-cycle URAM latency):** every run used WLAT=2; the 1-cycle
  path was deliberately avoided. (Contrast proof on conv_816: WLAT=1 →
  mismatch 1,152,229, max|err|=252.)
- **Wrong / per-tensor bias map:** per-OC `bias.mem` lanes verified live (harness
  `[BSC]` taps printed correct per-OC bias for each dispatch); no systematic-low
  signature anywhere.
- **Drain depth (`ag_mac_done_d5`):** every dispatch reached `done=1` and drained
  all output pixels (largest: conv_816 = 12,544 out-pixels, conv_912 = 49 ×
  1280-OC).
- **Requant rounding:** per-OC `scale.mem` (mult[15:0]/shift[21:16], +HALF,
  INT8-saturate) reproduced the golden exactly — including the deepest case
  conv_912 (OC=1280, 5 OC-passes, IC>256 chunk-straddle).
- **IC>256 chunk-straddle** (IC = 384 / 576 / 960, ic_chunks 2–4) and
  **multi-OC-pass** (up to 5) both exercised and byte-exact.

## Engine-TOP integration (the assembled `nn2rtl_top_engine.v`)

**Date:** 2026-06-02. **Scope:** wiring the proven `shared_engine` into the
full engine-dispatched top `output/mobilenet-v2/rtl/nn2rtl_top_engine.v` (the
`<80%` FIT artifact). The dispatch-level proof above is *per-module in
isolation*; this section is about the assembled top that strings 34 dispatches
through shared activation/weight memories, 33 input loaders, 34 output bridges,
and a real `nn2rtl_scheduler`.

### CONFIG-CORRECT — BUG 1 found and fixed (param override)

The `shared_engine` module *definition* the mbv2 build uses is
`output/rtl/shared_engine_skeleton.v` (the **real** module is lines 31–679;
lines 708–891 are SKELETON STUB versions of `mac_array`/`requant_pipeline`/
`address_generator`/`config_register_block`/`bram_to_stream_bridge` that the
real build does NOT use — the real submodules are `output/rtl/engine/*.v`). Its
DEFAULT params are the **ResNet INT4** config: `WGT_W=4` (line 49),
`URAM_DATA_W=1024` (line 57), `MAX_IH/IW/OH/OW=14` (lines 40–43).
`WEIGHT_RD_LATENCY` is a `localparam=2` (line 354), correct for mbv2, not
overridable.

**BUG 1 (CRITICAL, FIXED):** the `shared_engine u_shared_engine (`
instantiation at `nn2rtl_top_engine.v` had NO parameter-override block, so it
inherited `WGT_W=4` / `URAM_DATA_W=1024`. Consequence:
the 2048-bit `engine_weight_rd_data` bus (built from the 8×256-bit URAM-bank
concat at lines 1237–1244) was silently truncated to 1024 bits
(`WIDTHTRUNC` at the port, `UNUSEDSIGNAL` on `[2047:1024]`), dropping URAM banks
4–7 (OC lanes 128–255), and `shared_engine` (`WGT_W=4`) reinterpreted the
surviving low 1024 bits as 256 INT4 nibbles instead of 128 INT8 bytes. The
engine datapath would be completely wrong despite the proven byte-exact maps.
`MAX_IH/IW/OH/OW=14` also undersized the engine for the 112×112 dispatches
(0/1 = node_conv_814/816).

**Fix (on disk, `nn2rtl_top_engine.v` lines 2167–2176), verbatim matching the
proven iso harness `tb/engine_iso_wrap_mbv2.v`:**

```verilog
shared_engine #(
    .WGT_W(8),
    .URAM_DATA_W(2048),
    .MAX_IC(2048),
    .MAX_OC(2048),
    .MAX_IH(112),
    .MAX_IW(112),
    .MAX_OH(112),
    .MAX_OW(112)
) u_shared_engine (
```

`MAC_COUNT`, `BIAS_W`, `ACT_BUS_W` keep defaults, so the `bias_rd_data` /
`scale_rd_data` (8191:0 = 256×32) and `act_in/out` (2047:0) ports stay
width-correct (verified). `WGT_W=8` propagates to `mac_array` via
`.WGT_W(WGT_W)` (skeleton line 645), so every lane extracts a full INT8 byte.

### What static checks CANNOT validate (only an engine-top e2e value sim will)

The lint/elaborate pass proves config + widths + module resolution. It does
NOT exercise dynamic dataflow. Open items, in priority order:

1. **Final-stage flat-bus contract-width mismatch (10 SELRANGE, HIGH RISK,
   pre-existing, NOT a param bug).** `nn2rtl_top_engine.v` lines
   893/934/1006/1047/1205 extract **768 / 1280 bits from 256-bit** signals
   (e.g. line 893 `node_conv_880_data_out[767:0]` from a 256-bit data_out;
   lines 1006/1047/1205 take `[1279:0]`). These are the documented
   tiled-streaming ↔ flat-bus final-stage contract mismatch (768→256 /
   256→4096 / 1280→256) that `build_top_wrapper` never reconciled
   (`memory/project_mobilenet_u250_status`). They are present verbatim in
   `nn2rtl_top_engine.pre_skipwire.v`, so they are NOT a regression of the
   param fix. They will surface as **wrong values** in e2e value sim (out-of-
   range bits read as 0), not as an elaboration failure, and are a
   CONTRACT-LEVEL final-stage issue, not fixable by a surgical override.
2. **Dispatch timing / scheduler sequencing.** Scheduler hardcodes
   `LAST_DISPATCH=6'd33` (34 dispatches 0..33). `current_drain_complete =
   all_drain[sched_dispatch_idx]` and `current_loaded =
   all_loaded[sched_dispatch_idx]`. The dispatch advance/stall handshake,
   `spatial_stall` serialization with the spatial chain, and per-dispatch
   IC/OC/base ROM consumption are untested at the top level.
3. **Loader / bridge dataflow + BRAM arbitration.** 33 of 34 input loaders
   present (`u_ldr_node_conv_*` + `stream_to_act_bram_bridge BUS_W=256`); 34
   output bridges (slots 0..33). **Dispatch 21 = node_conv_876 has NO input
   loader and `all_loaded[21]=1'b1` hardwired** ("bank already resident" patch)
   — a SEQUENCING ASSUMPTION (876's input bank must be pre-populated by the
   prior dispatch) that only e2e value sim can confirm. Shared
   activation-BRAM read/write arbitration between engine, loaders, and bridges
   is likewise untested dynamically.

### Engine-top e2e value runner — STATUS: does NOT exist; must be built

There is **no** engine-top value runner. `scripts/run_mbv2_top_value.ts` is
hardcoded to the **all-spatial** path and cannot exercise the engine top:

- It builds `--top-module nn2rtl_top` and collects sources from `rtl/` only —
  but `nn2rtl_top.v` and `nn2rtl_top_engine.v` BOTH define `module nn2rtl_top`,
  so the engine top can never be in that source set without a name collision.
- It **injects an inactive engine+scheduler STUB** (`ENGINE_SCHED_STUB`,
  ~lines 88–214) with `engine_busy=0` / `spatial_stall=0`, which forces
  `spatial_run = ~(engine_busy | spatial_stall)` permanently high — i.e. it
  deliberately runs the pure all-spatial dataflow with the engine disabled.

**To build an engine-top value runner (concrete recipe):**

   `run_mbv2_top_value.ts`) that (a) sources `nn2rtl_top_engine.v` and
   **excludes** `nn2rtl_top.v` + `nn2rtl_top_engine.pre_skipwire.v`; (b) drops
   the `ENGINE_SCHED_STUB`; (c) adds the REAL engine source set: trimmed
   `shared_engine_skeleton.v` (lines 1–679 only — exclude the stub tail),
