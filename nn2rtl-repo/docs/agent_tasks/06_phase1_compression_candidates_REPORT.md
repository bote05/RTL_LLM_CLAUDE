## Observations worth flagging

- **`node_conv_290` has Fmax 429.9 MHz** while its 4 siblings (296, 294, 286, 300) all hit ~334 MHz. Same shape, different Fmax. Worth a quick look — possibly an auto-promoted reference variant that picked up a structurally faster pattern during earlier work.
- **The 6 BRAM-heavy modules at 105k LUT + 171 BRAM18 each** all run at exactly 218.7 MHz. Same template instantiated 6 times. Compressing the template would propagate 6×; the auto-promote infrastructure should fire if Phase 1 ever targets any one of them.
